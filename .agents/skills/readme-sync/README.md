# readme-sync

Keep README.md aligned with project goals and current state.

## What it does

Update README.md so it reflects what project meant to do, what it does now, and what user should know to use or extend it.

## When to use

Use this skill when user want README.md refreshed after code, scope, or workflow change.

## What to include

- Project goal and purpose.
- Current feature set.
- Important assumptions or constraints.
- Usage or setup steps if needed.
- Current state, not stale future plan.

## Rule

Keep README.md accurate, current, and readable. Do not leave goals or state out of sync with code.
