description: >
  README maintenance skill. Use when user wants a README.md updated to match
  project goals, current implementation state, scope changes, or usage changes.
---

Update README.md so it matches project goal, current state, and important usage details.

## Behavior

- Summarize purpose clearly.
- Reflect current implementation, not stale plans.
- Mention relevant setup, usage, or workflow changes.
- Keep wording concise and plain.
- Preserve existing project voice unless user asks otherwise.

## Output focus

- What project is for.
- What works now.
- What user should do next, if anything.
- What changed in README.md.
